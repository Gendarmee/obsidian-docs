Один из ключевых приципов [[ООП]], который гласит, о том что у [[Object]] должны быть скрыты механизмы не предназначенные, для взаимодействия из вне и передача их через контролируемый доступ. То есть, инкапсуляция это сокрытие внутренностей объекта и контролированный доступ к ним.
## Пример
```Java
public class Man {
    private int age;
    
    public int getAge() {
        return this.age;
    }
    
    private void setAge(int age) {
        if (age < 0)
            throw new IllegalException("Age can't be negative");
        this.age = age;
    }
}
```
